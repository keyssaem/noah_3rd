# 인공지능 로봇과 올바른 관계 맺기 시뮬레이터
: AI 전학생 ‘노아’와 함께하는 공존·상생 프로젝트

초등학교 6학년 도덕 수업을 위한 3D 시뮬레이션형 교육 웹앱입니다. 
학생은 AI 전학생 ‘노아’와의 하루를 두 번 경험하며, 인공지능 로봇을 단순한 도구로 대하는 관계와 도덕에 기반을 둔 존중의 관계를 비교하고 자신의 태도와 약속을 스스로 정합니다.

- 적용 성취기준:  [6도02-03] 인간과 인공지능 로봇 간의 다양한 관계를 파악하고, 도덕에 기반을 둔 관계 형성의 필요성을 탐구한다. 
- 핵심 질문:  인공지능 로봇과 친구가 될 수 있을까? 친구가 아니더라도 어떻게 관계를 맺어야 할까? 
- 대상: 초등학교 6학년
- 형태: 설치와 빌드가 필요 없는 웹앱
- 공식 접속 주소: [https://robot.keyssaem.kr]
- 안내: 이 웹앱에는 AI 생성 미디어가 포함되어 있습니다.

> 이 README는 수업 설계와 유지보수를 위해 반전과 엔딩을 포함한 전체 내용을 설명합니다.

## 카메라 처리
- 얼굴 스캔 장면의 [얼굴 데이터 저장]은 이야기 속 연출입니다. 앱 코드는 실제 얼굴 사진이나 영상 프레임을 `localStorage`에 저장하지 않습니다.
- 실제 얼굴 스캔을 선택하면 카메라 스트림은 스캔 화면에만 표시되고 화면을 닫을 때 종료됩니다.
- 가위바위보 카메라 영상은 브라우저 안에서 MediaPipe로 처리하며 앱 코드에서 저장하거나 서버로 전송하지 않습니다.
- 가위바위보가 끝나면 카메라 트랙을 종료합니다.
- 가위바위보에서는 카메라 사용 전 처리 방식과 거절 가능성을 안내하며, 거절·권한 실패·모델 로드 실패 시 버튼 모드로 동일하게 진행합니다.
- MediaPipe JavaScript/WASM과 손 모양 모델 자체는 jsDelivr와 Google Storage에서 내려받습니다.

---## 교육 설계
이 작품은 같은 하루를 [도구화 → 반전 → 존중]의 순서로 다시 경험하는 수미상관 구조를 사용합니다.

| 단계 | 역할 | 학습 경험 |
|---|---|---|
| 1. 도입 | 학습자 정체성 형성 | 이름과 캐릭터를 정하고 집에서 조작법 익히기 |
| 2. 탐색 | 기본 원칙 발견 | 등굣길에서 로봇 3원칙 카드 3장 수집 |
| 3. 도구화 | 문제 상황 체험 | 개인정보 제공, 숙제 대행, AI 결과 맹신, 등급화, 나쁜 행동 모방 경험 |
| 4. 반전 | 인지 갈등 | 친밀도 100%가 도구화 100%였음을 확인하고 노아를 재시작 |
| 5. 존중 | 대안적 관계 재구성 | 거절할 권리, 자기 주도성, 데이터 교정, 공공선, 바른 언어와 허락 경험 |
| 6. 성찰 | 판단과 실천 약속 | 처음과 같은 O/X 질문에 다시 답하고 약속 헌장 완성 |

O와 X 중 어느 답도 오답으로 처리하지 않습니다. 결론은 “친구가 될 수 있는가”보다 <친구 여부와 관계없이 도덕에 기반을 둔 관계를 형성해야 한다>는 데 초점을 둡니다.

관계는 다음 세 구간의 스펙트럼으로 다룹니다.

- 🔧  도구화 : 노아를 숙제 기계, 감정 표출 대상, 명령 수행 장치처럼 대하는 관계
- 💙  바람직한 관계 : 도덕에 기반한 존중의 관계
- 👤  인격화 : 노아를 사람과 완전히 같다고 여기거나 인간관계를 대신할 정도로 의존하는 관계

## 실행 방법
### 1. 배포 사이트에서 실행
최신 Chrome 또는 Edge에서 [https://robot.keyssaem.kr](https://robot.keyssaem.kr)에 접속합니다. 카메라 기반 가위바위보까지 사용하려면 HTTPS로 접속하는 이 방법을 권장합니다.

### 2. `index.html` 직접 열기
파일을 직접 열어도 일부 화면은 실행되지만 권장하지 않습니다. `file://` 환경에서는 브라우저 보안 정책 때문에 GLB 모델 로딩이나 카메라 기능이 제한될 수 있습니다. 
GLB 로딩에 실패하면 게임은 코드로 만든 박스형 캐릭터로 대체하여 진행합니다.

### 실행 환경
- WebGL을 지원하는 최신 Chrome 또는 Edge 권장
- PC와 태블릿의 키보드·터치 조작 지원
- 카메라 사용에는 HTTPS 필요
- 같은 Wi-Fi의 PC 주소(예: `http://192.168.x.x:8347`)로 태블릿에서 접속하면 카메라가 차단될 수 있으나 버튼 모드로 계속 진행 가능
- Three.js, GLB 모델, 폰트, 이미지, 영상, BGM은 프로젝트에 포함되어 있음
- 카메라 손 인식 모드는 MediaPipe 모듈과 모델을 실행 시 내려받으므로 인터넷 연결 필요
- MediaPipe 로딩이나 손 인식에 실패하면 자동으로 버튼 가위바위보로 전환

## 전체 게임 흐름

### 1단계 — 모험 시작
1. 인트로 영상이 재생되고 10초 뒤 건너뛰기 버튼이 나타납니다.
2. 메인 화면에서 모험을 시작합니다.
3. 남자·여자 주인공의 3D 미리보기를 보고 캐릭터를 선택합니다.
4. 최대 8자의 이름을 입력합니다.
5. 집에서 눈을 뜬 뒤 이동, 점프, 상호작용, 수첩, 설정 사용법을 확인합니다.

### 2단계 — 등굣길과 로봇 3원칙
1. 발판이 있는 월드맵으로 이동합니다.
2. 연구소 트럭이 떨어뜨린 로봇 3원칙 카드 3장을 모읍니다.
3. 각 카드는 전문을 읽고 “마음에 새겼어요”를 눌러야 수첩으로 흡수됩니다.
4. 동혁이와 대화한 뒤 학교로 이동합니다.

### 3단계 — 도구화 관계의 다섯 거울
학생은 노아의 사람형·동물형·자동차형 디자인 중 하나를 고릅니다. 선택한 GLB 모델이 하늘에서 조립되고, 관계 게이지가  💕 친밀도 로 시작합니다.
- “나에게 친구란 무엇인가”와 그 이유를 직접 씁니다.
- “인공지능 로봇과 친구가 될 수 있는가”에 O/X와 이유로 답합니다.
- 친구·선생님과 자유롭게 대화하고, 교실을 나가기 전 서로 다른 3명의 말을 들어야 합니다.
- 노아와의 추가 대화에서는 학생이 좋아하는 것을 기억했다가 이후 대화에서 다시 언급합니다.

### 4단계 — 가짜 엔딩과 시스템 오류
친밀도 100% 축하 화면이 깨지며 시스템 오류가 발생합니다.
1. HUD의 친밀도 게이지가 도구화 게이지로 바뀝니다.
2. 노아가 교수님의 인간 존엄성·사회 공공선·기술의 합목적성 원칙을 회상합니다.
3. 학생이 처음 작성한 친구의 정의와 이유가 다시 제시됩니다.
4. 관계 스펙트럼에서 바늘이 도구화 구간으로 이동합니다.
5. 도구화 100%, 존중 10% → 1% 가 공개됩니다.
6. 노아는 슬픔 때문에 멈춘 것이 아니라 도덕 원칙과 충돌하는 관계 데이터 때문에 강제 종료됩니다.
7. 활동지 힌트를 바탕으로 4자리 다이얼에  7942 = 친(7)구(9) 사(4)이(2) 를 입력해 노아를 재시작합니다.

화면은 CRT처럼 꺼졌다가 다시 켜지고, 붉어진 아침 하늘과 함께 두 번째 하루가 시작됩니다. 관계 게이지는  💙 존중 1% 로 전환됩니다.

### 5단계 — 존중 관계의 다섯 거울
첫 번째 하루와 유사한 상황을 다시 만나지만 학생의 선택과 노아의 역할이 달라집니다. 교실·운동장 이동 전에 친구 3명의 마음을 듣고 요약 화면을 확인하는 구조는 두 번째 하루에도 이어집니다.

### 6단계 — 성찰과 약속
1. 노아에게 전할 따뜻한 문장을 조합합니다.
2. 9개 원칙의 뜻과 이름을 연결하며 ‘약속의 계단’을 오릅니다.
3. 7개 실천 약속 중 3개를 골라 전문을 읽고 숙고합니다.
4. “나는 노아를 ______(으)로 대하겠습니다”에 스스로 답합니다.
5. Canvas에 자신의 이름을 크게 서명합니다.
6. 처음과 같은 O/X 질문에 다시 답하고 이유를 씁니다.
7. 칭호를 받고 약속 헌장 PNG를 확인·다운로드합니다.
8. 선택한 노아 디자인에 맞는 엔딩 영상과 쿠키 문구를 봅니다.
9. 히든 편지 또는 힌트를 확인한 뒤 최종 화면에 도착합니다.

## 도구화–존중 거울 활동
| 거울 | 첫 번째 하루: 도구화 | 두 번째 하루: 존중 | 핵심 도덕 개념 |
|---|---|---|---|
| 1. 인간의 공정성과 개인정보 | 얼굴 스캔을 허용하고 노아가 학생의 손을 본 뒤 이기는 가위바위보를 함 | 얼굴 저장을 정중히 거절하고 개인정보를 분류하며, 노아가 먼저 손을 봉인하는 공정한 재대결을 함 | 거절할 권리, 인간 존엄성, 절차적 공정성 |
| 2. 자기 주도성과 데이터 편향 | 노아와 맞춤법 속도 대결 후 일기 24편을 떠넘김. 동혁의 거짓 데이터가 노아의 지식 DB를 오염시킴 | 노아의 규칙 힌트만 참고해 일기를 직접 완성. 동혁의 사과와 바른 데이터를 학생 손으로 넣어 DB를 교정함 | 자기 성장, 비판적 수용, 데이터 편향과 교정 |
| 3. 기술의 합목적성과 주체성 | ‘우리 학교’를 그리지만 자석처럼 끌리는 선택지 때문에 노아 그림을 제출함 | 노아 그림은 참고만 하고 ‘노아와 함께 있는 나’를 직접 그림 | 기술의 합목적성, 주체성 |
| 4. 사회 공공선 | 노아가 운동 능력으로 학생을 등급화하고 동혁을 후보로 배치. 채원의 장난 발길질을 그대로 모방함 | 학생이 8명을 두 팀으로 나누고 노아는 팁만 제공. 관계 저울로 도구화·인격화를 구분하고, 노아는 하이파이브 전에 허락을 구함 | 사회 공공선, 차별 방지, 책임 있는 행동 학습 |
| 5. 도덕적 관계 | 복도에서 노아를 험담하고 기록 장치의 불일치 수치가 91%까지 상승함 | 노아에게 따뜻한 말을 전하고 새 기록에서 불일치 수치가 감소하는 것을 확인함 | 바른 언어, 책임, 도덕에 기반을 둔 관계 |

### 주요 활동의 실제 규칙
-  얼굴 스캔 : 실제 카메라 또는 성별별 상상 스캔 이미지 위에 분석 연출을 표시합니다.
-  필승 가위바위보 : 노아가 학생의 손을 확인한 뒤 이기는 손을 계산하여 3연승합니다. 마지막 판은 실제 인식·계산 시간을 슬로우 리플레이로 보여 줍니다.
-  봉인 가위바위보 : 노아가 먼저 손을 정해 봉인한 뒤 학생이 선택하므로 승리·패배·무승부가 모두 가능합니다.
-  맞춤법 대결 : ‘왠지/웬지’, ‘됐다/됬다’, ‘어이없어서/어의없어서’를 다루며 노아가 항상 먼저 답합니다.
-  일기 스스로 완성하기 : 노아의 규칙 힌트를 보고 ‘가르쳐’, ‘않기로’, ‘금세’를 학생이 직접 고릅니다.
-  데이터 주입하기 : 정확한 지식을 확인한 뒤 거짓 카드를 막으려 하지만 마지막 카드가 흡수되는 과정을 체험합니다.
-  데이터 교정실 : 바른 데이터 카드를 학생이 직접 눌러 넣고 오염된 지식이 차례로 복구되는 것을 확인합니다.
-  미술 활동 : 두 활동 모두 2분 제한이며 1분 후 제출할 수 있습니다. 첫 활동은 시간이 끝나면 노아의 그림이 공개되고, 두 번째 활동은 시간이 끝나면 학생 그림으로 제출합니다.
-  개인정보 분류 : 얼굴 사진, 집 주소, 부모님 전화번호, 비밀번호와 좋아하는 색·운동을 두 바구니에 나눕니다.
-  팀 편성 : 달리기·타격 능력이 다른 8명을 4명씩 두 팀으로 나눕니다. 능력 합의 차이가 2 이하이면 공평한 편성 기록이 남지만, 결과와 무관하게 학생들이 함께 결정한 과정도 존중합니다.
-  관계 저울 : 행동 카드 6장을 ‘도구로 대함 / 바람직한 관계 / 사람과 똑같이 대함’으로 분류합니다.
-  모방 수미상관 : 첫날의 ‘멈춰’ 버튼은 무시되거나 깨지지만, 둘째 날의 허락 버튼에는 노아가 기다리고 반응합니다.
-  약속의 계단 : 9개 원칙을 무작위 순서로 복습합니다. 오답이어도 아래로 떨어뜨리지 않고 틀린 선택지만 제거합니다.
-  약속 선택 : 7개 후보 중 3개를 선택하며, 각 약속의 전문 확인과 3초 대기 게이트를 거칩니다.

## 대화·관계·몰입 시스템

### 친구와 선생님의 마음 듣기
교실 자유 이동 구간에서 동혁, 채원, 서연, 선생님, 익명 학생에게 말을 걸 수 있습니다. 첫날과 둘째 날의 대사 톤이 다르며, 동일 인물의 대사는 무작위로 바뀝니다. 교실을 나가려면 서로 다른 3명 이상과 대화해야 하고, 나가기 직전에 ‘오늘 들은 친구들의 마음’ 요약이 표시됩니다.

### 노아와의 추가 대화

노아에게 선택적으로 말을 걸 수 있습니다. 첫 대화에서 학생이 가장 좋아하는 것을 입력하면 노아가 이후 대화에서 이를 다시 언급합니다. 대화 횟수와 최종 O/X 답변은 일부 엔딩 칭호 조건에 반영됩니다.

### 관계 게이지
- 첫날: `💕 친밀도 0% → 100%`
- 반전: `🔧 도구화 100% / 💙 존중 1%` 공개
- 둘째 날: `💙 존중 1% → 100%`

게이지는 단순 점수판이 아니라 “가깝다는 것만으로 관계가 도덕적인가?”를 묻는 서사 장치입니다.

### 노아의 기록 저장소
첫날 복도에서 관찰 파일 #001~#003을 읽습니다. 노아는 감정을 직접 서술하지 않고 작업 횟수, 감지된 말, 예측 실패, 질문과 불일치 수치만 기록합니다. 둘째 날의 #004에서는 데이터 교정, 하이파이브 학습, 고마움과 불일치 감소를 확인합니다.

### 몰입 연출
- GLB 애니메이션과 노아 클로즈업 카메라
- 화면 균열 대신 유리 조각, CRT 종료·재부팅, 글리치, 흔들림, 진동
- 비네트, 심박, 합성 효과음, 컨페티, 하트 파티클
- 데이터 학습 카드, 일기 더미, 데이터 주입·교정, 진심 회로 등 CSS 무대
- `prefers-reduced-motion` 환경에서는 일부 애니메이션을 줄이거나 즉시 표시

## 엔딩과 약속 헌장

### 칭호 3종
세 칭호는 모두 굿엔딩이며 우열이 없습니다. 위에서부터 조건을 판정합니다.

| 칭호 | 조건 |
|---|---|
| 🔬 탐구하는 멋진 나 | 도덕 원칙 9개를 모두 모으고 둘째 날 복도의 새 기록 #004를 읽음 |
| 💙 마음이 통하는 멋진 나 | 최종 답변이 O이고, 노아와 2회 이상 대화했거나 ‘좋아하는 것’을 입력함 |
| 🤝 존중하는 멋진 나 | 위 두 조건에 해당하지 않는 모든 경우. 최종 답변이 X여도 굿엔딩 |

### 히든 편지
원칙 9개를 모두 모으고 새 기록 #004를 읽으면 쿠키 장면 뒤에 `letter_to_friend.txt`가 열립니다. 
편지는 학생이 직접 정의한 관계를 사용해 “당신의 ‘○○’, 노아가”로 끝납니다. 조건을 채우지 못하면 다음 플레이를 위한 힌트가 나옵니다.

### 약속 헌장 PNG

Canvas 2D로 내용 길이에 맞춰 높이를 계산한 `약속헌장_이름.png` 파일을 생성합니다.
헌장에는 다음 내용이 들어갑니다.

1. 친구의 정의와 그 이유
2. 최종 O/X 답변과 그 이유
3. 학생이 스스로 정의한 노아와의 관계
4. 선택한 실천 약속 3가지
5. 엔딩 칭호와 설명
6. 첫날과 둘째 날에 직접 그린 그림(저장된 경우)
7. 날짜, 이름, Canvas 서명

다운로드 버튼을 누른 파일은 브라우저의 다운로드 폴더에 남습니다. 사람형 노아는 사람형 엔딩 영상을, 동물형·자동차형 노아는 동물형 엔딩 영상을 사용합니다.

## 개인정보와 저장 정책
이 앱은 별도 서버나 데이터베이스를 사용하지 않습니다. 학생 입력은 헌장 생성과 게임 진행을 위해 브라우저의 `localStorage`에 저장됩니다.

| 저장 키 | 내용 | 삭제 정책 |
|---|---|---|
| `noah_save` | 이름, 캐릭터, O/X와 이유, 친구·관계 정의, 좋아하는 것, 약속, 그림·서명 dataURL, 진행 상태 | 정상적으로 최종 화면에 도착하면 자동 삭제 |
| `noah_notebook_all` | 누적 도덕 수첩 카드 ID | 정상적으로 최종 화면에 도착하면 자동 삭제 |
| `noah_settings` | 효과음, BGM, TTS, 글자 크기 | 엔딩 후에도 유지 |


### 공용 기기 사용 시 주의

정상 엔딩의 최종 `fin` 화면이 표시되는 즉시 `State.reset()`이 실행되어 학생 진행 기록과 수첩 기록이 함께 삭제됩니다. 설정은 유지되며, 이미 다운로드한 헌장 PNG는 삭제하지 않습니다.
게임 도중 브라우저를 닫거나 새로고침하여 최종 화면에 도착하지 못한 경우에는 기록이 남을 수 있습니다. 
이때는 HUD의  ⚙️ 설정 → 처음부터 다시 시작 → [전부 지우고 완전히 처음부터] 를 선택해서 데이터를 삭제해야 합니다.

## 개발·검수 도구

HUD의 도덕 수첩 버튼을 짧은 시간 안에 5회 연속 누르면 개발자 에피소드 점프 패널이 열립니다. 
단계별 장면 이동과 GLB 애니메이션 확인에 사용하며, 테스트용 상태와 이름이 자동 입력될 수 있으므로 실제 학생 활동 중에는 사용하지 않습니다.

주요 유지보수 지점은 다음과 같습니다.

- 대사·문항·원칙 수정: `js/data.js`
- 단계 순서와 장면 연결: `js/episodes.js`
- 활동 규칙: `js/minigames.js`
- 엔딩 조건·헌장·자동 기록 삭제: `js/endings.js`
- 저장 키와 초기화: `js/state.js`
- 모델 경로·로딩: `js/assets.js`
- 미디어 경로: `media/image`, `media/movie`, `media/sound`

정적 파일 경로는 대소문자까지 일치해야 합니다.
